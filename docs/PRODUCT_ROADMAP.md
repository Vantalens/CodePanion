# RemindAI 产品路线

## 产品定位

RemindAI 是一个本地优先的个人 AI 工作流中控台，用于统一接入、查看、提醒和接管本机上的多个 AI 开发任务，并逐步演进为可编排、可复用的本地 AI 工作流操作台。

它服务的是一个高频使用多个 AI 编程工具的个人开发者，而不是团队协作平台。

## 核心原则

1. **本地优先**：默认把数据、状态和操作留在本机。
2. **个人优先**：先把一个人同时驾驭多个 AI 任务这件事做到足够好。
3. **先控制，后编排**：先解决“看得见、接得住、回得去”，再解决“能组织、能复用、能自动化”。
4. **不做团队化扩张**：不把多用户、权限、共享空间和企业协作作为产品目标。

## 阶段 1：个人本地控制台

### 目标

让一个人能够从一个界面稳定掌控本机上的多个 AI 任务。

### 关键能力

- 统一接入 CLI/PTTY、Codex Desktop、VS Code 和外部适配器
- 汇总任务运行、等待输入、完成和失败状态
- 提供桌面提醒、GUI 时间线和上下文查看
- 支持从统一界面回复可接管的任务
- 保持本地低占用、低干扰和清晰的来源边界

### 成功标准

- 多个任务同时运行时，用户无需在多个窗口之间来回找状态
- 任何等待输入的任务都能被及时发现
- 用户可以从一个界面判断“现在最需要处理什么”
- 核心链路具备稳定的自动化验证

## 阶段 2：本地 AI 工作流操作台

### 目标

让一个人不只能够监控 AI，还能够系统地组织 AI 完成复杂工作。

### 关键能力

- 工作流模板
- 多步骤任务编排
- 跨工具协作
- 任务结果归档
- 常用流程复用
- 对本地任务历史的搜索、筛选和回放

### 成功标准

- 常见任务可以被沉淀为可复用流程
- 不同 AI 工具能在同一条本地工作流中协同
- 用户能从“观察任务”升级到“设计任务”
- 系统仍然保持本地优先和个人使用友好

## 明确不做

- 不做团队版
- 不做多用户协作
- 不做权限、共享空间、审批流和企业审计
- 不把 RemindAI 发展成一个替代 Codex、Claude Code 或 VS Code 的聊天客户端

## 当前开发优先级

1. 稳定阶段 1 的基础链路
2. 补齐测试、状态持久化和长期运行稳定性
3. 提升多源接入质量，而不是只增加适配器数量
4. 在真实个人使用数据足够后，再启动阶段 2 能力建设
