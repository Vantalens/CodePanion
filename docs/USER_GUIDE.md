# CodePanion 用户指南

本文档面向实际使用 CodePanion 的用户，说明产品定位、核心界面和基本操作方式。

## 1. 产品说明

CodePanion 是一个本地优先、供应商中立、面向跨软件 / 跨窗口 / 跨项目场景的多任务操作平台。

它的核心目标不是替代 IDE、终端或 AI 工具本身，而是将分散在不同软件和不同窗口中的任务汇聚到一个统一的图形工作台中，帮助用户更高效地发现阻塞点、处理等待输入、跟进失败任务，并完成必要的任务级操作。

## 2. 适用场景

CodePanion 适合以下场景：

- 同时在多个 VS Code 窗口中推进不同项目
- 同时运行多个 Codex、Claude Code、CLI 或其他本地任务
- 需要及时发现等待输入、审批、失败和完成状态
- 希望减少频繁切换窗口带来的注意力中断

## 3. 界面结构

CodePanion 当前界面由四个主要区域组成。

### 3.1 左侧导航栏

用于切换不同任务视图：

- `全部`
- `等待`
- `运行中`
- `失败`
- `稍后`
- `产物`

### 3.2 任务列表

用于展示当前可管理的任务队列。

任务会根据处理优先级排序，重点状态会优先显示。用户无需打开任务详情，也可以快速判断当前是否有需要立即处理的任务。

### 3.3 任务工作台

这是当前任务的主操作区域，用于展示：

- 当前任务标题
- 来源与能力层级
- 当前状态
- 任务聚焦信息
- 主消息内容
- 等待输入或失败摘要

### 3.4 任务详情与代码产物区

右侧区域用于展示：

- 当前任务来源
- 能力边界
- 数据边界
- 可执行操作
- 任务转交与责任状态
- 代码产物或关键输出片段

## 4. 任务状态说明

CodePanion 当前使用以下任务状态：

| 状态 | 含义 |
|---|---|
| 等待我 | 当前任务正在等待用户输入或确认 |
| 失败 | 当前任务执行失败，需要排查 |
| 需审阅 | 当前任务已有输出结果，建议人工审阅 |
| 运行中 | 当前任务仍在执行 |
| 完成 | 当前任务已正常结束 |
| 稍后处理 | 当前任务已被延后处理 |
| 已归档 | 当前任务已移出主队列 |

## 5. 基本操作

### 5.1 查看任务

在左侧任务列表中选择任意任务，任务工作台会同步展示该任务的主要内容、当前状态与下一步建议。

### 5.2 处理等待输入

当任务进入 `等待我` 状态时，界面会优先提示该任务。若当前来源支持写回，用户可以直接在界面中回复或继续执行。

### 5.3 查看失败任务

当任务失败时，主视图会展示失败摘要，并提供结构化诊断上下文复制能力，便于继续排查或转交其他工具处理。

### 5.4 管理任务

当前版本支持以下基础任务管理动作：

- `置顶任务`
- `稍后处理`
- `归档任务`
- `恢复任务`
- `调整优先级`
- `手动上移 / 下移`

已稍后或已归档的任务会移出主队列，并进入对应视图，避免干扰当前待处理任务。

### 5.5 批量处理任务

在当前视图中进入 `批量选择` 后，可以同时对多项任务执行：

- `批量恢复`
- `批量归档`
- `批量稍后`
- `批量置顶`
- `批量优先级调整`

批量动作适合在同一个项目或同一波并行任务需要集中整理时使用。

### 5.6 转交任务

右侧任务详情区提供 `任务转交` 面板。当前版本支持：

- 为 `通用 / Codex / Claude Code / OpenCode` 生成标准化交接包
- 导出完整上下文
- 导出更适合直接粘贴到目标工具中的转交指令
- 在当前任务上直接 `启动并转交`
- 在目标工具开始接手后标记 `已接手`
- 在需要收回处理权时标记 `已回流`
- 在转交关系结束后 `清除转交`

如果本机存在对应目标工具入口，CodePanion 会直接创建一个新的接力会话；如果目标工具不可用，则会回退到本地交接会话，避免转交流程中断。

界面会明确展示当前任务是否处于 `待转交 / 已转交 / 已回流` 状态、当前目标工具，以及已经建立的接力会话。

当前版本已经支持最小可执行的任务接力闭环：启动转交、建立接力会话、记录责任状态，并在接力会话结束后把原任务自动回流到当前队列。

## 6. 任务来源与能力层级

CodePanion 支持从不同来源接入任务，但不同来源的能力并不完全相同。界面会明确标注能力层级，避免用户误判。

| 层级 | 含义 |
|---|---|
| L1 | 进程识别 |
| L1/L2 | 弱接入 |
| L2 | 只读事件 |
| L2/L3 | 事件可回 |
| L3 | 可回写会话 |
| L4 | 工作流编排 |

这意味着：

- 并非所有来源都支持直接回复
- 并非所有来源都支持完整接管
- 弱接入来源不会被包装成深度集成来源

## 7. 典型使用流程

一个典型使用流程如下：

1. 启动 CodePanion 图形界面
2. 让多个工具或多个窗口中的任务正常运行
3. 在 CodePanion 中观察任务队列变化
4. 优先处理 `等待我` 和 `失败` 任务
5. 对暂时不处理的任务执行 `稍后` 或 `归档`
6. 在需要时复制诊断上下文、任务转交包，或切回原工具继续深度操作

## 8. 当前产品边界

CodePanion 当前不是：

- 完整 AI IDE
- 模型聊天客户端
- 通用个人 Agent
- 通用启动器
- 系统级进程监控器

它当前聚焦于：

> 将分散在多个软件、多个窗口和多个项目中的任务，统一收束到一个图形工作台中进行管理与处理。
