# CodePanion 产品定位契约

本文档根据 `D:\Owen\Documents\OneDrive\桌面\deep-research-report.md` 校准 CodePanion 的对外定位，用于约束 README、路线图、GUI 文案、包描述和后续功能规划，避免产品叙事与市面已有产品正面冲突。

## 定位声明

CodePanion 是一个本地优先、供应商中立的 AI 编程工作流中控台 / 控制平面。它不替代任何 AI IDE、CLI、插件或模型聊天产品，也不试图成为通用个人 Agent；它专注于把开发者在本机同时运行的多个 AI 编程工具的任务状态、等待输入、异常、结果与上下文统一收束到一个图形工作台里，并提供最小必要的接管、回复、切换与审计能力。

## 核心差异

| 相邻产品类型 | 容易撞车的说法 | CodePanion 的正确说法 |
|---|---|---|
| 通用个人 Agent / OpenClaw 类 | 全能个人助手、消息通道聚合、自动接管电脑 | 只面向本机 AI 编程任务，采集和回写都遵守显式边界 |
| 单厂 AI 工作台 / Codex / Claude Code Desktop | 多 Agent 工作台、替代某个厂商客户端 | 跨供应商、跨入口、跨形态的外层控制台 |
| AI IDE / Trae / Comate / CodeBuddy | 新一代 AI IDE、代码生成平台 | 不做编辑器和代码生成能力，回到原工具完成深度操作 |
| Raycast 类 launcher | 效率启动器、万能入口 | 快速入口只服务 AI 编程任务生命周期，不扩展成通用启动器 |
| Activity Monitor 类系统监控 | AI 进程监控器、资源观察器 | 面向任务单元的状态、等待输入、回复、切换和审计 |

## 做什么

- 汇聚本机 AI 编程工具来源、项目路径、窗口归属和能力层级。
- 优先呈现等待输入、失败、运行中、需审阅和完成状态。
- 在公开 API、CLI/PTTY、只读同步、VS Code 扩展和显式适配器范围内完成回复、跳转、通知与审计。
- 让用户知道“谁在运行、谁在等我、谁失败、我应该先回复谁、需要回到哪个原工具”。
- 将国产 AI IDE / 插件作为 L1/L2/L3 分层适配目标，而不是把存在识别包装成深度接管。

## 不做什么

- 不做通用个人 Agent、聊天聚合器、邮箱/日历/IM 总控或后台 cron 平台。
- 不做完整 AI IDE、模型聊天客户端、代码生成质量竞争或 token 二次分销。
- 不做 Raycast 式通用 launcher，也不把产品退化为系统进程监控器。
- 不读取账号、token、cookie、浏览器登录态、插件私有数据库、上游私有 API、全局屏幕内容或默认剪贴板内容。
- 不提前做团队协作、共享空间、权限审批平台或企业管理后台。

## MVP 焦点

0 到 1 的 MVP 只围绕“等待你处理的任务队列”成立：来源中心、任务状态看板、等待输入检测、直接回复、原工具跳转、项目切换、通知中心、审计时间线和隐私边界说明。工作流模板、跨工具转派、指标面板和更深国产工具接入，应在这个闭环稳定后再推进。
